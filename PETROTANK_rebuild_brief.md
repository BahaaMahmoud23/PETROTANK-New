# PETROTANK Website Rebuild Brief

## Project Goal
Rebuild the PETROTANK website with a **new modern, professional UI** while preserving the **same real business content, same page structure, and same corporate credibility** from the original website.

This is **not** a startup-style redesign and **not** a random concept.
It is a **professional corporate rebuild** inspired by strong competitors in the tank storage / terminal / energy logistics space.

## Required Public Pages
- Home
- About Us
- Services
- Capabilities
- Certifications
- Contact

## Style Direction
The new design should be:
- modern
- premium
- light industrial corporate
- highly readable
- structured
- trustworthy
- visual but not flashy

### Competitive Inspiration Direction
Use the following inspiration mix:
- **Ocean Point Terminals** → homepage structure, strategic positioning, trust and infrastructure storytelling
- **Europort Terminal** → hero treatment, service block visuals, industrial image-led presentation
- **Buana Shipping & Marine** → multi-page corporate structure, contact and internal page organization

Do **not** copy these websites directly.
Only borrow the **layout quality, hierarchy, composition, and professionalism**.

## Brand / Theme Direction
Use the original PETROTANK logo as the brand anchor.

### Light Theme Palette
Use a clean premium light theme based on the logo colors:

- Primary Blue: `#355486`
- Secondary Blue: `#3D6C98`
- Teal Accent: `#5EABB3`
- Soft Teal: `#7ABCC2`
- Deep Navy Text: `#2E4A78`
- Light Background: `#F7FAFC`
- White Surface: `#FFFFFF`
- Light Border: `#D9E4EC`
- Muted Text: `#5E6E82`

### Usage Rules
- background should be light, not plain white everywhere
- text should be strong and readable
- blue is the main tone
- teal is the accent
- use subtle gradients only when elegant
- no random orange
- no heavy dark theme

## Typography
- English: Inter
- Arabic: Cairo

Use:
- strong but professional headings
- readable body text
- clear hierarchy
- balanced spacing

## Technical Requirements
Build with:
- Next.js App Router
- Tailwind CSS
- Framer Motion

Requirements:
- clean reusable components
- SEO-friendly structure
- responsive on desktop / tablet / mobile
- maintainable architecture
- content separated from presentation where useful

## Motion Direction
Use Framer Motion in a premium restrained way.

Allowed:
- subtle section reveals
- count-up metrics
- soft hover interactions
- elegant staggered entrances
- timeline animation in About page
- subtle map/form reveal in Contact page

Avoid:
- flashy animation
- playful motion
- heavy parallax
- anything that hurts readability or performance

## Content Source of Truth
Use the original PETROTANK content exactly in meaning.
You may rewrite only for clarity, hierarchy, and readability.
Do not invent fake claims, fake numbers, fake certificates, fake facilities, or fake contact data.
